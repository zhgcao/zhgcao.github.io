Git只获取部分目录的内容

发表于 2016-05-11

公司的开发从SVN切换到Git，在SVN下，可以很方便的只获取版本库中一个或多个目录的内容，但是Git的克隆，默认是直接拉取整个远程仓库，这样大量和自己无关的内容也会拉到本地，占用很多硬盘空间。

在网上搜了一下，Git在1.7版本后，已经支持只Checkout部分内容，这个功能叫做sparse checkout，用起来稍麻烦一点。

要用这个功能，先进入要放版本库的目录，在命令行执行几条命令：

第一条命令git init ems，先建立一个空的版本库。
第二条命令cd ems，进入创建的新的版本库的目录。
第三条命令git remote add origin ssh://<user>@<repository's url>，添加远程库的地址。
第四条命令git config core.sparsecheckout true，打开sparse checkout功能。
第五第六条命令echo "path1/*" >> .git/info/sparse-checkout，添加2个目录到checkout的列表。路径是版本库下的相对路径，也可以用文本编辑器编辑这个文件。
第七条命令git pull origin master，拉取远程的 master 分支，也可以拉其他分支。
如果只拉取最近一次的变更，忽略以前的变更记录，在拉取时可以加参数depth，如git pull --depth=1 origin master

如果以后修改了 .git/info/sparse-checkout，增加或删除部分目录，可以执行如下命令重新Checkout

其实这样Checkout，仅仅是其他的文件不显示在工作区，整个分支的变更记录仍然会被拉到本地，这点和SVN还是不同的，不过就算这样，也能节省不少空间了。
